# Vision Report: Explainable Hybrid GraphRAG for Conversational Recommendation (Master's Thesis)

## 1. Objectives and Strategic Vision (The Vision)
The target architecture represents a definitive shift away from legacy predictive recommendation algorithms toward a dynamic, context-aware conversational agent. The system addresses the fundamental limitations of pure LLM implementations (hallucinations, lack of deterministic grounding) and traditional engines (cold-start problem) by synthesizing **Multi-Agent orchestration**, **Hybrid GraphRAG retrieval**, and **explicit Knowledge Graph reasoning**.

The strategic vision of this project explicitly abandons the computationally prohibitive goal of training custom graph neural networks on massive e-commerce datasets. Instead, it optimizes for Explainability, Groundedness, and Conversational Fluidity, ensuring every recommendation generated by the agent is transparent, verifiable, and logically sound.

## 2. User Experience (UX) and System Behavior
* **Natural Conversation**: A chat-based interface (e.g., built with Chainlit/Streamlit). The user does not feel like they are manually querying a database; instead, they interact with an expert advisor by describing implicit needs (e.g., "I'm looking for a laptop for computer science studies that can handle newer games").
* **Active Clarification**: If a query is too general, the system does not guess. The agent autonomously determines when to seek clarification, asking targeted questions to refine the user's constraints and update their profile.
* **Explainability (XAI) & Data Provenance**: Every presented recommendation transcends black-box suggestions. It contains a fact-based justification explicitly derived from the Knowledge Graph's topology, connecting the user's historical preferences directly to the recommended target.

## 3. Core Engine Logic
The project moves away from rigid processing pipelines and pure Text-to-Cypher generation in favor of a modern **Multi-Agent System (MAS) + Hybrid GraphRAG** paradigm.

1. **Multi-Agent Orchestration (Router & Critic)**: 
    * **Orchestrator**: The central cognitive engine that classifies user intent to dictate the dialogue policy. It autonomously decides whether to seek clarification, update the persistent user profile, or initiate a retrieval sequence.
    * **Critic**: A specialized agent that evaluates retrieved candidates against the nuanced, soft constraints of the user's prompt prior to final generation, ensuring strict adherence to the user's implicit desires.
2. **Entity-Based Memory Architecture (MemoCRS)**: Moving beyond the noisy concatenation of raw dialogue history, the system utilizes a User-Specific Memory bank. It stores discrete, extracted entities and user attitudes, allowing for targeted semantic retrieval of relevant historical context across sessions without context-window degradation.
3. **Hybrid Semantic-Structural Retrieval**: 
    * **Vector Search**: Natural language intents are embedded into dense vectors for rapid Approximate Nearest Neighbor (ANN) search via Neo4j Vector Indexes.
    * **Structural Filtering**: Semantic results are simultaneously and deterministically filtered using hard Cypher constraints (e.g., price boundaries, brand exclusions) parsed dynamically by the Orchestrator.
4. **Response Generation with Explicit Reasoning**: By injecting explicit graph reasoning pathways into the LLM's context window, the model is architecturally constrained to generate responses rooted in demonstrable data provenance.

## 4. Key Architectural Decisions & Scope Definition
To guarantee the successful execution and focus of the master's thesis, the project scope is rigorously defined:
* **Abandoning Custom Model Training**: The project consciously avoids the extreme computational burden of training custom Relational Graph Convolutional Networks (R-GCN, KGAT, LoRA). It focuses entirely on Prompt Engineering, RAG architecture, and high-quality pre-trained LLM embedding models applied to enriched textual representations of graph nodes.
* **Dual-Dataset Graph Construction (Scope Protection)**: The project utilizes a highly targeted, dual-dataset strategy. The modern **LLM-REDIAL dataset** will serve as the conversational and behavioral foundation, providing validated multi-turn dialogues. To construct the Knowledge Graph without unmanageable data engineering overhead, the system will use the **Amazon Reviews 2023 dataset** strictly as a metadata lookup. Only the items specifically present in the selected LLM-REDIAL domains will be extracted from the massive Amazon catalog to populate the graph's rich structural (Products, Brands, Categories) and lexical (Review Chunks) nodes.
* **Knowledge Graph as Grounding**: The system relies on the graph structure to completely prevent LLM hallucinations. The system cannot invent a product or its features.
* **Primary Academic Contribution**: The thesis will academically demonstrate how the integration of Hybrid GraphRAG retrieval and Multi-Agent verification significantly enhances the groundedness and user-perceived explainability of conversational recommendations compared to baseline LLM retrieval methodologies.

## 5. Metrics and Evaluation (How do we measure success?)
Evaluating conversational recommender systems (CRS) requires a dual-dimensional approach based on modern research standards:

### A. Recommendation Quality (System Dimension)
* **NDCG@K (Normalized Discounted Cumulative Gain)**: Evaluating the ranking quality delivered by the Hybrid Search mechanism.
* **Hit Rate (HR@K) / Recall@K**: Verifying if the target (correct) item is present in the top-K items retrieved from the database.

### B. Conversation Quality (LLM-as-a-Judge Evaluation)
We will use an advanced LLM model to evaluate the generated responses according to criteria from state-of-the-art CRS frameworks:
* **Coherence**: Does the system properly follow the user's intent and maintain a logical conversation flow?
* **Explainability**: The system's ability to provide clear, detailed, and verifiable justifications for *why* a given product fits the user's preferences based on explicit graph paths.
* **Groundedness**: Measuring the extent to which the response relies *exclusively* on data and relationships fetched from the Knowledge Graph (strictly penalizing attribute hallucinations).
* **Recoverability**: The system's ability to refine and correct its outputs based on negative user feedback or shifted preferences.
