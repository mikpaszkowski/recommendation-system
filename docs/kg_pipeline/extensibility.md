# Extensibility Notes

The foundational schema elevates `Review` events, `Attribute` facets, and `CoPurchaseSet` groupings to first-class nodes so that future commonsense augmentations can attach without remodeling core entities. Reserved labels (`OpinionPhrase`, `CommonsenseEntity`) and the `RELATES_TO`/`SUPPORTS` edge patterns create a dedicated lane for ingesting LLM-derived triples such as `<Product>-[:USED_FOR]-><Audience>` while preserving provenance (`sentiment`, `confidence`, `extraction_version`). Because attributes are normalised and reusable across products and reviews, newly discovered commonsense relations can be bridged to existing nodes instead of generating duplicates, enabling explainable, multi-hop traversals that mix explicit catalog data with implicit user narratives.

For representation learning, the schema offers a heterogeneous graph with typed edges (`WROTE`, `REVIEWS`, `HAS_ATTRIBUTE`, `BELONGS_TO_CATEGORY`, `BOUGHT_TOGETHER`, etc.), ideal for R-GCN, FastRP, or GraphSAGE pipelines. The ingestion script records `ingest_batch_id` and timestamp metadata, making it straightforward to project time-sliced subgraphs or maintain feature stores for nightly embedding jobs in Neo4j Graph Data Science. Furthermore, `Review` nodes host both structural edges and raw text, ensuring Mutual Information Maximisation (MIM) objectives or Graph-to-Text adapters (COMPASS, GraphRAG) can align token embeddings with graph neighbourhoods—the same nodes feed full-text indexes for retrieval while their neighbours supply structural hints for serialization. The explicit price buckets, variants, and co-purchase bundles collectively deliver the rationales an LLM requires to articulate grounded recommendations.
